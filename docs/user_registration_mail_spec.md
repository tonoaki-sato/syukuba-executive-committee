# ユーザー登録関連 メール送信仕様書

## 1. 概要
本システムにおけるユーザー登録フロー（新規会員申請の承認・却下、および管理者による直接のユーザー追加）に伴うメール通知機能の仕様を定義します。

現在、本システムでは実機によるSMTPメール送信処理は組み込まれておらず、**Laravelのログ機能 (`Log::info()`) を使用してシステムログ (`storage/logs/laravel.log`) に出力する方式（モック実装）**となっています。本番環境や外部送信を有効にする場合は、本仕様のログ出力処理を `Mail` ファサード等に置き換えて対応します。

---

## 2. 送信メール一覧

| 種類 | トリガー元 | 送信先 | 件名 | 主な内容 |
| :--- | :--- | :--- | :--- | :--- |
| **① 登録承認通知メール** | 仮会員の承認時 | 対象ユーザーのメールアドレス | `【重要】保土ケ谷宿場まつり実行委員会 登録承認およびログイン設定のご案内` | 承認完了の通知、および24時間有効なパスキー設定用URLの案内。 |
| **② 申請却下通知メール** | 仮会員の申請却下時 | 対象ユーザーのメールアドレス | `登録申請に関するご連絡` | 申請見送りの通知、詳細問い合わせ案内。 |
| **③ アカウント作成通知メール** | 管理者による会員直接追加時 | 追加されたユーザーのメールアドレス | `【重要】保土ケ谷宿場まつり実行委員会 アカウント作成のお知らせ` | アカウントが直接作成された通知、および24時間有効なパスキー設定用URLの案内。 |

## 3. 詳細仕様

### 3.1 登録承認通知メール (仮会員承認時)
管理者が仮登録会員（`status = temporary`）の申請を承認した際、システム上で24時間有効なワンタイムトークンが発行され、パスキー設定用のURLとともにログへ出力されます。

* **送信トリガー**: `AdminController@approveUser`
* **ログ識別ヘッダー**: `=== ユーザー承認通知メール ===`
* **メール設定値**:
  - **宛先 (To)**: `{ユーザーのメールアドレス}`
  - **件名 (Subject)**: `【重要】保土ケ谷宿場まつり実行委員会 登録承認およびログイン設定のご案内`
* **本文 (Body) テンプレート**:
  ```text
  {ユーザー名} 様

  実行委員会への登録が承認されました。以下のURLより、24時間以内にログイン用パスキー（指紋・顔認証）の設定を行ってください。
  パスキー設定URL: {パスキー設定用ワンタイムURL}
  ```
* **仕様上の注意点**:
  - パスキー設定用URLは `PasskeySession` に紐づく `token` をクエリに含みます（URL例: `https://{domain}/passkey/register/{token}`）。
  - このURL（およびトークン）の有効期限は**発行から24時間** (`expires_at` に `now()->addHours(24)` を格納) です。

---

### 3.2 申請却下通知メール (仮会員申請却下時)
管理者が仮登録会員（`status = temporary`）の申請を却下（ステータスを `rejected` に変更）した際にログへ出力されます。

* **送信トリガー**: `AdminController@rejectUser`
* **ログ識別ヘッダー**: `=== 登録申請却下メール ===`
* **メール設定値**:
  - **宛先 (To)**: `{ユーザーのメールアドレス}`
  - **件名 (Subject)**: `登録申請に関するご連絡`
* **本文 (Body) テンプレート**:
  ```text
  {ユーザー名} 様

  誠に恐れ入りますが、実行委員会への登録申請は見送られました。詳細はお問い合わせください。
  ```

---

### 3.3 アカウント作成通知メール (管理者による会員追加時)
管理者が「会員追加」機能から新しいユーザーを直接登録（ステータス `active` で即時作成）した際、ログイン用のパスキー登録ワンタイムトークンを発行し、その設定URLとともにログへ出力されます。

* **送信トリガー**: `AdminController@storeUser`
* **ログ識別ヘッダー**: `=== ユーザー直接追加登録通知メール ===`
* **メール設定値**:
  - **宛先 (To)**: `{追加されたユーザーのメールアドレス}`
  - **件名 (Subject)**: `【重要】保土ケ谷宿場まつり実行委員会 アカウント作成のお知らせ`
* **本文 (Body) テンプレート**:
  ```text
  {追加されたユーザー名} 様

  管理者によって実行委員会システムのアカウントが作成されました。
  以下のURLより、24時間以内にログイン用パスキー（指紋・顔認証）の設定を行ってください。
  パスキー設定URL: {パスキー設定用ワンタイムURL}
  ```
* **仕様上の注意点**:
  - 承認時と同様、ワンタイムトークンの有効期限は**発行から24時間**です。

## 4. 技術的な補足とログの出力イメージ

### ログの実体定義 (実装コード例)
システム内でメール送信がトリガーされた際、`AdminController` 内部では以下のように `Log::info()` が呼び出されます。

```php
Log::info("=== ユーザー承認通知メール ===");
Log::info("To: {$user->email}");
Log::info("Subject: 【重要】...ご案内");
Log::info("Body: ");
Log::info("{$user->name} 様\n...");
Log::info("==================================");
```

これにより、サーバー側の `storage/logs/laravel.log` に以下のように書き出されます。

```text
[2026-07-12 11:00:00] local.INFO: === ユーザー承認通知メール ===  
[2026-07-12 11:00:00] local.INFO: To: user@example.com  
[2026-07-12 11:00:00] local.INFO: Subject: 【重要】保土ケ谷宿場まつり実行委員会 登録承認およびログイン設定のご案内  
[2026-07-12 11:00:00] local.INFO: Body:   
[2026-07-12 11:00:00] local.INFO: テスト太郎 様
...  
[2026-07-12 11:00:00] local.INFO: パスキー設定URL: http://localhost/passkey/register/abcdef123456...  
[2026-07-12 11:00:00] local.INFO: ==================================  
```

### 将来的な実機送信への移行方針
将来的に本物のメールとして外部送信を行う場合は、以下の実装アプローチを取ります。
1. `php artisan make:mail UserApprovedMail` などで Mail 区分を生成する。
2. コントローラー側で `use Illuminate\Support\Facades\Mail;` をインポートする。
3. ログ出力部分（`Log::info(...)`）を以下のように書き換える。
   ```php
   Mail::to($user->email)->send(new UserApprovedMail($user, $registerUrl));
   ```
4. `.env` 内の `MAIL_MAILER` や `MAIL_HOST` などの SMTP 設定情報を本番サーバーの環境に合わせて設定する。
