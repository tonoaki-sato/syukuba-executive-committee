# 広告・協賛管理機能 仕様書

本仕様書は、保土ケ谷宿場まつり実行委員会における「広告・協賛」の申込、入金管理、新聞掲載用の入稿管理業務を効率化するためのシステム仕様について定義します。

---

## 1. 業務背景とシステム化の目的
毎年の「保土ケ谷宿場まつり」開催に向けて、地域店舗や企業から協賛金および宿場まつり新聞への広告掲載（広告枠）を募っています。
これまではFAXやメールによる申込書を紙やExcelで個別管理していましたが、以下の課題を解決するためシステム化を行います。

- **入金状況の可視化**: 誰が未入金であるかを一覧で把握し、督促や入金確認漏れを防ぐ。
- **新聞掲載への自動連動**: 申込日や入金日、入稿締切（広告: 8月7日、協賛: 8月15日）に応じた「新聞掲載対象者」の自動リストアップ。
- **広告原稿・テキスト情報の管理**: 広告掲載店舗ごとの入稿画像ファイルや、事務局が作成代行するテキスト情報を管理画面で一元管理する。

## 2. 協賛・広告の基本ルール（PDFより抽出）

### 2.1 協賛（Sponsorship）
- **単価**: 1口 5,000円（1口以上、上限なし）
- **申込締切（新聞掲載期限）**: **2026年8月15日(土)**
- **特典**: 上記期限までに申し込んだ場合、「保土ケ谷宿場まつり新聞」の協賛者一覧にお名前（氏名または企業・店舗名）が掲載される。

### 2.2 広告（Advertisement）
- **単価**: 1枠 10,000円（最大30枠まで）
- **入稿締切**: **2026年8月7日(金)**
- **広告枠サイズ (1枠)**: 縦 34mm × 横 82mm
- **特典**: 新聞に指定枠サイズで広告（ロゴ・画像・店舗紹介テキスト等）が掲載される。
- **入稿方法**:
  1. **オリジナル原稿（方法1）**: 申込者側で作成した JPEG または PNG 画像をそのまま使用する。
  2. **テキスト入稿（方法2）**: 事務局が以下の情報に基づき指定レイアウトで作成する。
     - 画像（写真またはロゴ等）
     - 店舗名
     - 記事の区画数（＝購入枠数）
     - キャッチコピー（18文字以内）
     - 紹介文（17文字×5行＝最大85文字）
     - 住所、営業時間、電話番号、定休日

### 2.3 振込先・問い合わせ先
- **振込先**: 横浜銀行 保土ケ谷支店 普通 6108908 保土ケ谷宿場まつり実行委員会
- **事務局担当者**: 山道 在明（ひかり歯科）

## 3. データベース設計（データモデル）

### 3.1 `sponsorships` テーブル (協賛・広告申込データ)

| 物理カラム名 | 論理名 | 型 | 制約 | 補足説明 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | ID | `bigint` | Primary Key, Auto Increment | |
| `fiscal_year` | 対象年度 | `integer` | NOT NULL | 例: 2026 (年度別の集計用) |
| `type` | 申込種別 | `string` | NOT NULL | `advertisement` (広告) / `sponsorship` (協賛) |
| `name` | 氏名・企業名・店舗名 | `string` | NOT NULL | 新聞掲載時の掲載名義としても使用 |
| `contact_name` | 担当者氏名 | `string` | NOT NULL | |
| `contact_info` | 連絡先 (電話/メール) | `string` | NOT NULL | |
| `website_url` | HPリンク先URL | `string` | Nullable | 宿場まつりHPのリンク集掲載用 |
| `quantity` | 口数 / 枠数 | `integer` | NOT NULL, Default: 1 | 広告は「枠数」、協賛は「口数」 |
| `unit_price` | 単価 | `integer` | NOT NULL | 広告: 10,000円 / 協賛: 5,000円 |
| `total_amount` | 請求総額 | `integer` | NOT NULL | 自動計算: `quantity` × `unit_price` |
| `payment_status` | 入金ステータス | `string` | NOT NULL | `unpaid` (未入金) / `paid` (入金済) |
| `paid_at` | 入金日 | `date` | Nullable | 入金確認した日付 |

### 3.2 `advertisement_assets` テーブル (広告原稿データ)

| 物理カラム名 | 論理名 | 型 | 制約 | 補足説明 |
| :--- | :--- | :--- | :--- | :--- |

| `sponsorship_id` | 申込ID | `bigint` | Foreign Key (sponsorships.id), Delete Cascade | |

| `submission_status`| 入稿ステータス | `string` | NOT NULL | `pending` (未着) / `submitted` (入稿済) / `approved` (確認・承認済) |
| `image_path` | 入稿画像ファイルパス | `string` | Nullable | オリジナル画像、または紹介用写真・ロゴの保存パス |
| `catchphrase` | キャッチコピー | `string(18)` | Nullable | 最大18文字制限 |
| `body` | 紹介文 | `text` | Nullable | 最大17文字×5行制限 (85文字) |
| `address` | 住所 | `string` | Nullable | |
| `business_hours` | 営業時間 | `string` | Nullable | |
| `holiday` | 定休日 | `string` | Nullable | |
| `phone_number` | 電話番号 | `string` | Nullable | |

## 4. 主要機能要件

### 4.1 事務局・管理者向け管理画面
1. **広告・協賛ダッシュボード**
   - 当年度の合計申込金額、入金済総額、未入金総額、広告枠の消化数（目標枠数に対する進捗）を表示。
   - 口数・枠数別の集計グラフ。
2. **申込一覧・検索・編集画面**
   - 申込情報を一覧表示。氏名、種別（広告/協賛）、入金状況、入稿状況でのフィルタ検索。
   - 各レコードの「入金ステータス」「入金日」を一覧からダブルクリックまたはトグルでインライン更新できるクイック操作機能。
   - 申込用紙（紙・FAX・メール）から手動で代理登録するための新規登録フォーム。
3. **新聞掲載用エクスポート機能**
   - 指定した日付（例: 広告締切 8/7、協賛締切 8/15）までに「入金完了」または「申込完了」している対象者を抽出し、新聞用の名前リストや入稿用データをCSV/Excel形式で出力する。
4. **広告原稿レビュー・承認機能**
   - 広告申込者の入稿ステータス管理、入稿された画像ファイルのプレビュー表示、テキスト情報（キャッチコピー等）の確認、および「承認（印刷用データ確定）」へのステータス変更。

### 4.2 外部公開（申込者向け）機能要件（将来の拡張またはオプション）
- **オンライン申込フォーム**
   - 外部から企業・店舗が直接「広告」または「協賛」をオンラインで申し込めるフォーム。
   - 申込完了時に振込先情報が自動記載された確認メールを送付。
   - 広告申込の場合、そのままオリジナル画像ファイル（PNG/JPEG）をアップロードできる機能。
