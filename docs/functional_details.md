# 保土ケ谷宿場まつり実行委員会 実務管理総合システム 機能詳細設計書

本書は、システム仕様書（[system_specifications.md](file:///opt/project/syukuba-executive-committee/docs/system_specifications.md)）に基づき、初期フェーズ（会員管理、会議体管理、認証、システム共通基盤）で実装する具体的な機能の詳細を定義する。

## 1. 共通 UI/UX 仕様

### 1.1 画面レイアウト
- **フレームワーク**: Bootstrap (CDN経由で読み込み)
- **テーマ**: レスポンス対応（スマートフォン/PC両対応）。まつりの活気と伝統を感じさせる落ち着いた和モダンテイスト（エンジ色や濃紺などをベースとした配色）。
- **共通ヘッダー**: 
  - ロゴ（保土ケ谷宿場まつり実行委員会）
  - 現在の操作対象「開催年（年度）」の表示および切り替えドロップダウン（管理者のみ変更可能、一般会員は参照のみ）
  - ログイン中のユーザー名・所属
  - マイページ / ログアウトリンク
- **ナビゲーション**: スマートフォンではドロワーメニュー、PCではサイドバーまたは上部ナビゲーションを提供。

### 1.2 フロントエンド開発規制
- **CSP（コンテンツセキュリティポリシー）準拠**:
  - すべてのHTML内の `onclick="..."` や `<script>...</script>`、`<style>...</style>` 等のインライン記述を禁止する。
  - JavaScriptによるイベントリスナーの登録（`addEventListener`）およびスタイル適用は、すべて外部ファイル（`/public/js/app.js` 等）に記述して呼び出す。

## 2. 認証・アカウント管理機能

### F2.1 ログイン画面
- **対象**: すべてのユーザー
- **機能概要**: パスキー優先（Passkey-First）および条件付きUI（Conditional UI）設計に基づく認証を行う。
- **認証フロー**:
  1. **ID入力欄フォーカス時 (Conditional UI)**:
     - ブラウザのオートフィル機能と連携し、登録済みのパスキー候補をキーボード上部等にドロップダウンで自動提案する。
     - ユーザーが候補を選択してデバイスの生体認証（指紋・顔認証等）またはPIN入力を行うだけで、即座にログインが完了する。
  2. **手動入力時の挙動**:
     - メールアドレスを入力して「次へ」ボタンを押下した際、サーバー側で該当アカウントに対するアクティブなパスキーの登録有無を判別する。
     - **パスキー登録あり**: ブラウザのパスキー認証ダイアログ（WebAuthn）を自動起動し、生体認証を要求する。パスキー認証をスキップまたは失敗した場合は、パスワード入力欄を表示する。
     - **パスキー登録なし**: 即座にパスワード入力欄を表示する。
  3. **パスワード認証（フォールバック）**:
     - パスワード入力後、認証が一致した場合は、必要に応じて多要素認証（MFA）を要求し、検証完了後にログインさせる。

### F2.2 ユーザーパスキー（認証情報）管理機能
- **対象**: システム管理グループに所属するユーザーのみ
- **機能概要**: 一般ユーザー自身によるパスキーの自由な追加・削除は禁止されている。管理者が主体となってパスキーのライフサイクル（発行、一時無効化、削除、バックアップ登録）を管理する。
- **画面構成（管理画面内）**:
  - **ユーザー別パスキー一覧**: 指定したユーザーに紐づいている登録済みパスキー（登録日時、デバイス名、最終使用日時）を表示。
  - **登録用セッション（ワンタイムURL）の発行**: 対象ユーザーを指定して登録用ワンタイムURLを発行する。ユーザーは本URLへアクセスすることで、自身のデバイス（スマホ・PCなど）でパスキーの登録処理（公開鍵送信）を実行できる。
  - **パスキー無効化（削除）ボタン**: デバイス紛失時などに、該当のパスキーを管理画面から即座に無効化（削除）する。
  - **パスワード初期化機能**: パスキー紛失等でログインできないユーザーに対し、一時的なパスワードとMFA初期化を提供する。

### F2.3 一般ユーザー用パスワード自己変更機能
- **対象**: ログイン済みの一般会員・幹事
- **機能概要**: パスワードログインを利用しているユーザーが、自身のパスワードを安全に変更する。
- **画面構成（マイページ内）**:
  - 現在のパスワード、新しいパスワード、新しいパスワード（確認用）の入力エリア。

## 3. システム共通基盤・年度移行機能

### F3.1 開催年（年度）切り替え
- **対象**: 全ユーザー（一般会員は閲覧のみ、幹事・管理者は切替可能）
- **機能概要**: システム全体のアクティブな年度を切り替え、表示される全データをその年度のものに同期する。
- **動作仕様**:
  - ヘッダーのドロップダウンから年度（例: 「2026年度（第33回）」）を選択。
  - セッションまたはデータベースに選択された年度情報を保持し、画面遷移時もその年度のコンテキストを維持する。

### F3.2 新年度立ち上げ・データ引き継ぎ
- **対象**: 管理者
- **機能概要**: 新しいまつりの開催年に向けて、システム内の基本設定および会員情報を前年度から移行・作成する。
- **処理ステップ**:
  1. **新年度の登録**: 新規に作成する開催年（例: `2027`）を入力。
  2. **会員の引き継ぎ選択**:
     - 前年度（例: `2026`）の会員一覧がリスト表示される。
     - デフォルトで全員がチェックされた状態とし、退会予定のメンバーのチェックを外して移行対象を選択する。
     - 新年度における役職・会員属性（幹事、一般会員など）の仮設定を行う。
  3. **その他データのコピー**:
     - ござ市の出店ルール設定や備品マスタなどのマスタデータを前年度から複製（任意選択）。
  4. **実行処理**:
     - `comittee_users` などのテーブルにおいて、新しい年度に紐づくレコードを一括生成する。

## 4. 会員管理機能

### F4.1 メンバー（実行委員）一覧
- **対象**: ログイン済みの全ユーザー
- **機能概要**: 実行委員の情報を閲覧・検索する。
- **表示項目**: 氏名（かな）、会員種別（一般/幹事等）、所属（本業・得意分野）、紹介者名、LINEアカウント名、当年度の活動ステータス。
- **検索・フィルタリング**:
  - キーワード検索（氏名、本業、LINEアカウント名）
  - 種別（幹事、一般会員）での絞り込み
  - LINEのアカウント名から実名を逆引きするための検索

### F4.2 メンバー登録・編集
- **対象**: 幹事・管理者
- **機能概要**: 紹介制に基づいた新規メンバーの登録、および情報の変更を行う。
- **入力項目**:
  - **基本情報**: 氏名（必須）、かな（必須）、メールアドレス（ログインID）
  - **会員種別・属性**: 一般会員、幹事、その他の属性（複数選択可）
  - **出自情報**: 本業・職業（必須）、所属団体、得意分野（電気工事、デザイン、調理など）
  - **紹介関係**: 「紹介者」を既存の会員リストからドロップダウンで選択（必須、該当なしの場合は「自己応募」等を選択）
  - **LINE情報**: LINEアカウント名（LINE内表示名）、アイコンの特徴などのメモ（「この人誰？」防止用）
  - **ステータス**: 在籍 / 休会 / 除籍

### F4.3 除籍判定・アラート機能
- **対象**: 幹事・管理者
- **機能概要**: 活動実態のない幽霊会員を抽出し、除籍候補としてリストアップする。
- **判定ロジック（フィルタ条件）**:
  - 会議出席率が一定基準以下（例：過去3回連続で無断欠席）
  - LINEグループでのリアクション・発言が一定期間確認できない
  - まつり当日の参加登録がない
- **画面構成**:
  - 活動未確認メンバー一覧（最終確認日、欠席履歴等を可視化）。
  - 「一括注意喚起メール/連絡用テキスト生成」機能。
  - 管理者判断による「除籍手続き（ステータス変更）」実行ボタン。

### F4.4 新規会員登録機能（自主登録申請）
- **対象**: 未ログインの一般ユーザー（外部公開画面）
- **機能概要**: 新たに実行委員会に参加する希望者が、自ら会員情報を入力してシステム利用の申請（仮登録）を行う。
- **入力項目**:
  - 氏名（必須）、かな（必須）、メールアドレス（ログインID、必須）、仮パスワード。
  - 出自情報（本業・職業（必須）、所属団体、得意分野）。
  - 紹介者（既存会員から選択、またはフリー入力）。
  - LINEアカウント名（LINE内表示名、必須）。
- **処理仕様**:
  - 登録申請完了時点では、ユーザーの会員属性は **「仮会員」** となる。

### F4.5 仮会員の機能制限（アクセスロックアウト）
- **対象**: ステータスが「仮会員」のログインユーザー
- **動作仕様**:
  - **仮会員は、システム管理者に承認されるまで、システムの一切の機能を使用できない。**
  - ログインに成功した場合であっても、「管理者によるアカウント承認待ちです」というメッセージが書かれた簡易画面（メニューやナビゲーションのない閉鎖画面）のみが表示される。
  - マイページ、メンバー一覧、会議管理、各種拡張機能など、他すべてのURLへのアクセスはサーバーサイドで拒否（`403 Forbidden`）とする。

### F4.6 仮会員承認機能
- **対象**: システム管理グループに所属するユーザーのみ
- **機能概要**: 申請中の仮会員の情報を精査し、正式な会員として承認または却下（削除）を行う。
- **画面構成（管理画面内）**:
  - **仮会員申請一覧**: 申請中のユーザー（氏名、職業、紹介者、申請日時）をリスト表示。
  - **詳細確認・編集**: 申請内容の不備や、紹介者との関係性、初期の役割（一般会員、幹事等）を調整。
  - **承認アクション**: 承認ボタンを押下すると、会員ステータスが「仮会員」から「正式会員（一般会員等）」に昇格し、各種機能の利用権限が付与される。**この際、操作を行ったシステム管理者のユーザーID（承認者ID）および承認日時がシステムに記録される。**承認完了の通知メールを自動送信する。
  - **却下アクション**: 却下（削除）ボタンを押下すると、申請データが削除される。

## 5. 会議体管理・LINE連携機能

### F5.1 会議スケジュール・出欠管理
- **対象**: ログイン済みの全ユーザー（一般会員は出欠回答のみ、幹事は会議作成・編集が可能）
- **機能概要**: 月1回の幹事会、総会、および臨時部会のスケジュール調整と出欠確認を行う。
- **画面構成**:
  - 会議カレンダー / 一覧。
  - 出欠登録フォーム（出席 / 欠席 / 未定、および欠席理由・連絡事項の入力）。
  - 幹事用：出欠集計ダッシュボード（出席率の自動計算）。

### F5.2 会議案内 LINE投稿テキスト生成
- **対象**: 幹事・管理者
- **機能概要**: 会議の開催案内をLINEグループへスムーズに投稿するための文章を自動生成する。
- **出力内容**:
  - 会議名、日時、場所（またはオンラインURL）、アジェンダ、システム上の出欠登録ページのURL。
  - 「コピー」ボタン（クリップボードにコピー）を設置し、LINEアプリへの貼り付けを容易にする。

### F5.3 議事録登録とホワイトボード内容文章化
- **対象**: ログイン済みの全ユーザー（閲覧）、幹事（登録・編集）
- **機能概要**: 会議の議事録を作成し、ホワイトボードの画像内容を文章化して保存する。
- **画面構成**:
  - **基本情報**: 会議日、出席者、決定事項。
  - **画像アップロードエリア**: スマホ等で撮影したホワイトボードの画像を複数枚アップロード可能。
  - **議事録本文（テキストエリア）**: 画像を見ながら文字起こし・文章入力を行うエリア。
  - **LINE共有用テキスト生成**: 決定事項や議事録の要約をLINEグループに報告するためのテキストを生成・コピーするボタン。

## 6. 将来拡張機能のシステム設計（データ連携方針）

本システムは将来的に以下のモジュールを追加するため、テーブル設計時に `comittee_` 接頭辞を使用し、`開催年（年度）ID` を外部キーとして持つ設計を徹底する。

| モジュール名 | 主要機能・データ要件 | 共通基盤（年度管理）との連携 |
| :--- | :--- | :--- |
| **ござ市管理** | 出店者情報、出店区画割り、出店料入金ステータス | 年度ごとに新規区画・出店者を管理。過去年度の出店履歴を参照。 |
| **会計管理** | 予算設定、経費精算、領収書画像管理、収支報告 | 年度ごとの予算枠と決算処理。複数年度を跨ぐ繰越金の管理。 |
| **通行止め管理** | 道路使用許可申請、通行止めエリアマップ、警備員配置 | 年度ごとの警備計画・申請書類。前年度のマップデータを複製利用。 |
| **衛生管理** | 検便実施リスト、食品取扱届出、保健所提出ステータス | 年度ごとの検便検査結果（有効期限管理）、届出状況の確認。 |
| **備品管理** | 備品台帳、保管場所、貸出履歴、破損・補充管理 | 年度ごとの備品割当。貸出履歴は年度ごとにアーカイブ。 |
| **安全管理** | 警備体制計画、救護所配置、防災計画、緊急連絡網 | 年度ごとの警備シフト・連絡網。前年度の体制図を引き継ぎ可能。 |
| **文書管理** | 申請書テンプレート、提出済書類、各種マニュアル | 年度ごとの申請文書フォルダ。版管理と年度アーカイブ。 |
| **イベント管理** | 催し物スケジュール、ステージ出演者、スタッフシフト | 年度ごとのプログラムタイムラインと担当者配置。 |
