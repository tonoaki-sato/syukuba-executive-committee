# ユーザー利便性向上のためのサイト導線（リンク構成）改善提案書

## 1. 目的
本システム（保土ケ谷宿場まつり実行委員会 実務管理システム）において、一般会員から管理者まで、すべてのユーザーが迷うことなく必要な機能にアクセスできるよう、サイト全体のナビゲーション（導線）およびリンク構成の改善を提案します。

---

## 2. 現状の課題分析

1. **ポータル依存の高いアクセス経路**:
   - 「備品管理」「安全管理（警備計画）」「文書管理（組織図等）」といった重要な実務ツールがヘッダーナビゲーションに存在せず、ポータルの下部エリアにしかリンクがありません。ユーザーは毎回ポータルを開き、下までスクロールしてアクセスする必要があります。
2. **ヘッダーリンクの冗長性と機能の偏り**:
   - 左端のロゴリンクと「ポータル」メニューが重複しており、ヘッダーのスペースが有効活用されていません。
   - 一般会員に表示されるメニューが「ポータル」「会議管理」の2つのみであり、自分が何を実行・確認できるのかが直感的に分かりにくい構成となっています。
3. **「ござ市管理」の類似機能名の混在（幹事・管理者向け）**:
   - メニュー内に「出店場所配置」と「出店場所地図配置」という酷似した名前のリンクが並んでおり、それぞれ「一覧表での区画割り」「地図上へのピン配置（グラフィカル操作）」という目的の違いが分かりにくくなっています。
4. **「戻る（パンくずリスト）」導線の不足**:
   - 「会議詳細 → 会議一覧」や「出店応募詳細 → 応募一覧」などの下層ページから親ページへ戻るリンクが「ブラウザの戻るボタン」やヘッダーの再クリックに依存しており、一貫したパンくずリスト（Breadcrumbs）が不足しています。
5. **ポータル画面のボタンデザインの一貫性欠如（追加予定のモジュール）**:
   - ポータル下部の「追加予定の実務管理モジュール」において、他のモジュール（備品管理、安全管理等）のリンクボタンは `btn-outline-primary`（アウトライン型）で統一されていますが、「ござ市管理」のボタンだけが `btn-primary`（塗りつぶしボタン）で実装されており、デザインの一貫性を欠いています。

---

## 3. 改善提案内容

### ① ヘッダーナビゲーションの再整理（グローバル導線の強化）
全ユーザーが頻繁に確認する実務コンテンツを、ロールに応じた最適な見せ方でヘッダーに配置します。

* **一般会員（`general` ロール）のヘッダー表示構成（案）**:
  1. **ポータル**（ロゴリンクと統合し「ポータル」の個別テキストリンクは削除）
  2. **会議・出欠**（「会議管理」から、ユーザー目線に合わせた「会議・出欠」に改称）
  3. **組織・文書一覧**（組織図や警備計画書を格納する「文書管理」へ直結）
  4. **備品管理**（各自の貸出状況や機材在庫を確認できるページへ直結）

* **幹事・管理者向けのヘッダー表示構成（案）**:
  一般向けメニューに加え、既存の「ござ市管理」「管理者メニュー」を引き続きドロップダウンで提供しますが、以下の文言および階層変更を行います。

| 現状のメニュー名 | 提案メニュー名 | 理由・効果 |
| :--- | :--- | :--- |
| **会議管理** | **会議・出欠管理** | 出欠の登録・集計が主目的であることを分かりやすく提示。 |
| **出店場所配置** | **出店区画割り (一覧)** | テーブル形式での「ブース・番号の割り当て」であることを明確化。 |
| **出店場所地図配置** | **会場マップ配置 (地図)** | マップ画像上にピンを配置するグラフィカルな画面であることを明確化。 |

---

### ② ポータル（ダッシュボード）の情報整理とクイックリンク
ポータル画面（`dashboard.blade.php`）を、ユーザーがログインした瞬間に「今日行うべきこと」がわかる管制塔（コントロールパネル）として再整理します。

* **直近の予定の強調**:
  - 「1ヶ月以内の直近の会議予定」の上部に、自分が「未回答」の会議がある場合にのみ目立つ形で**「未回答の出欠が X 件あります。回答してください。」**というアラートバナーを動的に表示。
* **実務アクションカードの拡張**:
  - 「会議予定・出欠」や「プロファイル」のショートカットの横に、一般会員向けとして「組織図・安全計画書を読む」「備品の貸出申請を行う」カードを大きく配置。
* **追加モジュールのボタンデザインの統一**:
  - ござ市管理のボタンも、他のモジュール（備品管理・安全管理など）と同様にアウトラインボタン（`btn-outline-primary`）に統一します。
  - ボタンの文言も他の「管理画面へ」「文書一覧」と整合させ、「管理画面へ」（一般会員は「地図閲覧へ」）に統一し、全体の統一感を持たせます。

---

### ③ パンくずリスト（Breadcrumbs）の導入
すべての詳細画面や登録・編集画面のコンテンツ最上部に、親階層へ戻るためのパンくずリストを配置します。

* **実装イメージ（会議詳細画面の場合）**:
  ```text
  ポータル  >  会議一覧  >  第3回幹事会 (詳細)
  ```
  これにより、迷子になるのを防ぐとともに、数階層上の親カテゴリにワンクリックで戻れるようになります。

---

### ④ 「戻る」ボタンの配置ルール統一
現在、画面ごとに「戻る」「一覧へ」「キャンセル」などのボタン配置やスタイル（`btn-outline-secondary` 等）がバラバラになっているため、配置ルールを統一します。

* **配置の標準化ルール**:
  - **右下配置**: 「保存する」「決定する」などのポジティブアクションボタン（`btn-primary`）を右端に配置。
  - **左下またはアクションボタンの左隣配置**: 「キャンセル」「戻る」などの遷移ボタン（`btn-outline-secondary`）をその左側に配置。
  - フォーム送信を伴わない単純な詳細画面では、コンテンツ最上部（またはヘッダーのすぐ下）に `← 会議一覧に戻る` というシンプルなテキストリンクを配置。

---

## 4. 期待される効果
* **アクセシビリティの向上**:
  - 一般ユーザーが「組織図」や「警備計画」のPDFを探してポータル下部までスクロールする手間がゼロになります。
* **オペレーションミスの削減**:
  - 幹事が「区画の割り当て」と「地図へのピン配置」の画面を間違えて操作する混乱を防ぎます。
* **回答率の向上**:
  - 未回答の出欠予定がポータル最上部で強調されるため、会議の出欠回答漏れが少なくなります。
