# 保土ケ谷宿場まつり実行委員会 実務管理総合システム メイン通り地図管理機能 仕様書

本書は、システム仕様書（[system_specifications.md](../system_common/system_specifications.md)）および「ござ市管理仕様書」（[gozaichi_specifications.md](../gozaichi/gozaichi_specifications.md)）に基づき、イベント会場であるメイン通りのデジタル配置地図管理機能の要件・業務フロー・データモデル・画面設計の詳細を定義する。

---

## 1. 機能概要

イベントの円滑な運営のため、メイン通り沿いの出店ブース、付帯設備、および保健所指導に基づく給水設備等の配置状況を視覚的に管理・共有する地図機能を提供する。
ベースとなる地図画像（大久保さん送付の配置用平面図）の上に、ドラッグ＆ドロップ等のインタラクティブな操作で各種オブジェクト（マーカー）を配置・管理できるWebマップアプリケーションとして構築する。

### 1.1 導入効果
- **配置調整の効率化**: 毎年変化する出店位置や設備配置の調整・割り当て作業をWeb上でドラッグ＆ドロップで行い、変更内容を即座に共有。
- **保健所指導の順守**: 給水（手洗い）設備からの距離制限（20m以内）を地図上でビジュアル化し、配置ミスの防止。
- **住民対応（クレーム）の可視化**: 過去の住民のクレーム箇所や配置禁止箇所の周知徹底。
- **セクション別表示**: ごみ箱、音響スピーカー、三角コーン等のセクション別配置をレイヤー切り替えでクリアに表示。

## 2. 権限・アクセス制御

地図に関する全操作（配置、追加、編集、削除）は、幹事およびシステム管理者に制限する。

| ロール名 | 値 | 地図機能における権限 |
| :--- | :--- | :--- |
| **幹事** | `kanji` | 地図の閲覧、マーカーの追加・編集・削除、レイヤー切り替え、PDF出力等のすべてのマップ管理機能 |
| **システム管理** | `admin` | 幹事のすべての権限に加え、地図設定、座標基準点等のシステム管理者限定操作 |
| **一般会員** | `general` | 地図の閲覧、表示レイヤー切り替え、およびPDF出力のみ（編集・ドラッグ配置は不可） |

## 3. 地図画面の基本構成とレイヤー設計

### 3.1 背景（ベースマップ）
- 大久保さん送付の配置図（[大久保さん送付0923-2.pdf](./大久保さん送付0923-2.pdf)）を画像化したもの（PNGまたはSVG）を背景として重ね合わせて表示する。
- 建物や道路などの物理配置は固定で動かない。
- 地図画像はレスポンシブに対応させ、拡大・縮小（ズーム）およびドラッグ移動（パン）が可能とする。

### 3.2 表示レイヤー（セクション切り替え）
画面上のコントロールパネルより、以下のレイヤーを個別に表示・非表示（重ね合わせ）切り替えができるようにする。

| レイヤー名 | マーカー種別 | マーカー表現（アイコン・カラーなど） |
| :--- | :--- | :--- |
| **ござ市出店者** | 出店応募データ | - 火器使用飲食: **赤色** の店舗型ピン<br>- 火器なし食品: **緑色** の店舗型ピン<br>- 一般物販（その他）: **黒色** の店舗型ピン |
| **付帯設備** | 共通備品配置 | - ごみ箱: ダストボックスのアイコン<br>- 音響スピーカー: スピーカーのアイコン<br>- トイレ: WCマークのアイコン<br>- 三角コーン（通行止め）: コーンのアイコン |
| **保健所対応** | 給水（手洗い）設備 | - 給水設備: **青色** の蛇口アイコン<br>- カバーエリア: 蛇口を中心に、調理場所から歩行距離20mに相当する**青い点線の円（カバー円）**を描画 |
| **イベント・企業** | ブース配置 | - イベントブース（防犯コーナー等）: **オレンジ色** のテントマーク<br>- 企業ブース（寺井ブース等）: **茶色** のテントマーク |
| **クレーム（注意）** | 過去苦情・警告点 | - クレーム発生箇所: **黄色** または **赤色** の「⚠️」警告アイコン。<br>- 範囲指定: 影響範囲を薄い赤色サークルで描画 |

## 4. 機能詳細と操作仕様

### 4.1 マーカー配置・スナップ（吸い付き）効果
- **ドラッグ＆ドロップ配置**: ツールボックスから任意のマーカー（ごみ箱、スピーカー等）を地図上の任意の場所にドラッグ＆ドロップして配置する。
- **スナップ効果**: 地図の通り（メイン道路）に沿ってマーカーが綺麗に配置されるよう、道路の中央線や区画に沿った**ガイド線（スナップガイド）**を内部座標で定義する。マーカーがガイド線の付近（例: 30ピクセル以内）にドラッグされた際、自動的にガイド線上に吸い付くように配置補正を行う。
- **ござ市出店者配置の連動**:
  - 「ござ市管理」で当選している出店者をリストからドラッグし、地図上の区画位置に配置できる。
  - 配置時に、出店者の `spot_code`（区画コード）や `x_position` / `y_position` 座標がデータベースに連動して保存される。

### 4.2 マーカー詳細モーダル
配置されたマーカーをクリックするとモーダルが表示され、以下の操作および確認が行える。
- **ござ市出店者の場合**:
  - 屋号名、代表者氏名、出店タイプ（火気/食品の有無）、連絡先、支払い状況の確認。
  - 区画コード（`spot_code`）の変更および配置解除（削除）。
- **付帯設備・イベント・給水設備の場合**:
  - 名称、管理責任セクション、メモの編集・更新・削除。
- **注意・クレームマーカーの場合**:
  - 過去のクレーム内容（例: 「金子クリーニング前は配置禁止」「脇道をふさがないこと」）や対応方法の閲覧・編集。

### 4.3 誤配置の防止（インテリジェント警告）
1. **クレームエリア接近警告**:
   - 注意・クレームマーカー（または指定された配置禁止範囲）の一定範囲内に出店者マーカーがドラッグされた際、マップ上で該当エリアを赤く光らせ、「注意：金子クリーニング前は配置制限エリアです」といったツールチップ警告を即座に表示する。
2. **給水設備範囲外警告**:
   - 火器使用飲食（赤）または食品取扱（緑）の出店者が配置された際、どの給水設備（青）のカバー円（半径20m以内）にも入っていない場合、出店マーカーの周囲に「給水設備から20m以上離れています」と警告マーク（警告アイコン）を自動描画する。

### 4.4 PDF出力機能
- 画面上で現在アクティブにしているレイヤーやマーカーがそのまま保持された形で、地図全体をPDFとして出力・ダウンロードできる機能を提供する。
- ブラウザのプリント機能（`window.print()`）による印刷用CSSの適用、またはフロントエンドでのキャンバスデータ変換（`html2canvas` ＋ `jsPDF`等）を利用し、A3サイズ等に最適化された配置図PDFを生成する。

## 5. データベース設計

共通のテーブル接頭辞 `comittee_` を使用する。

### 5.1 `comittee_map_markers` テーブル（地図マーカー・オブジェクト管理）

地図上のすべての配置マーカー（ござ市、付帯設備、給水設備、クレーム情報など）を本テーブルで一元管理する。

| カラム名 | データ型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | BigInt | PK, Auto Increment | マーカーID |
| `fiscal_year` | SmallInt | Not Null | 年度（共通年度管理と連携、例: `2026`） |
| `marker_type` | Enum | Not Null | マーカー大分類（`'gozaichi'`, `'facility'`, `'water'`, `'event'`, `'claim'`） |
| `sub_type` | Varchar(50) | Nullable | 小分類（`'trash'`, `'speaker'`, `'toilet'`, `'cone'`, `'member_B'`, `'temp_booth'` 等） |
| `x_position` | Double | Not Null | 地図上のX座標（％指定、0.00〜100.00） |
| `y_position` | Double | Not Null | 地図上のY座標（％指定、0.00〜100.00） |
| `name` | Varchar(100) | Not Null | マーカー名称（店舗名や設備名、例: 「金子クリーニング前警告」） |
| `description` | Text | Nullable | 詳細説明、メモ、クレーム内容など |
| `application_id` | BigInt | Nullable, FK | `comittee_gozaichi_applications.id` への外部キー（ござ市出店者連携用。カスケード削除） |
| `created_at` | Timestamp | Nullable | レコード作成日時 |
| `updated_at` | Timestamp | Nullable | レコード更新日時 |

#### リレーションシップ:
- `GozaichiApplication` モデルに対して `hasOne` または `belongsTo` リレーションを組み、地図座標とござ市申込情報が相互参照できるようにする。

## 6. ルーティング設計

既存のルートグループ内に、以下の地図管理用ルートを追加する。幹事・管理者のみに制限（ミドルウェア `gozaichi`）を適用する。

```
Route::middleware(['auth', 'approved', 'gozaichi'])->prefix('goza/map')->name('goza.map.')->group(function () {
    // 地図編集・閲覧画面
    GET  /goza/map                 → GozaichiMapController@index    (一般会員・幹事・管理者共通閲覧 / 幹事・管理者のみ編集操作可能)
    
    // マーカーAPI (非同期操作用)
    GET  /goza/map/markers         → GozaichiMapController@getMarkers
    POST /goza/map/markers         → GozaichiMapController@storeMarker
    PUT  /goza/map/markers/{id}    → GozaichiMapController@updateMarker
    DELETE /goza/map/markers/{id} → GozaichiMapController@deleteMarker
    
    // PDFダウンロード用ルート
    GET  /goza/map/pdf             → GozaichiMapController@exportPdf
});
```

## 7. 画面設計イメージと操作性

- **ドラッグ操作パネル**: 画面左側に「追加オブジェクト」のパネルを配置。ごみ箱、スピーカーなどのアイコンをドラッグできる。
- **配置済みリスト**: 右側に「未配置の出店者リスト」をアコーディオン形式で表示。ドラッグして配置するとリストから消え、地図上にピンが立つ。
- **レイヤーセレクター**: 右上にフローティングトグルメニューを配置。「ごみ箱」「スピーカー」「給水」「クレーム警告」等のチェックボックスで重ね合わせを制御。
- **スナップ（吸い付き）エフェクト**: ドラッグ中に通りの付近にくると点線の「スナップガイド」が表示され、ガイド上にピタッと張り付くマイクロアニメーションを提供する。
- **注意エリア警告**: マーカーをクレーム発生点に近づけると、該当エリアが透過赤色の円で脈動（パルス）アニメーション表示され、注意喚起を行う。

## 8. ベースマップPDFアップロード・自動変換機能

システム管理者（`admin`）向けに、イベント会場の変更や店舗区画の基礎レイアウトが変更された際、最新の配置用平面図（PDF）をアップロードするだけでWeb地図の背景画像を自動更新・差し替える機能を提供する。

### 8.1 機能仕様
- **アップロード対象ファイル**:
  - 拡張子: `.pdf` (MIMEタイプ: `application/pdf`)
  - 最大ファイルサイズ: 10MB
- **アクセス権限**:
  - **システム管理者 (`admin`) 限定**。一般会員および幹事（ござ市担当）は実行できない（誤差し替えによるピン座標ズレやシステム全体への影響を防ぐため）。
- **変換処理仕様**:
  - アップロードされたPDFの1ページ目を自動でレンダリングし、横幅 1600px 相当のPNG画像として `public/images/map_base.png` に上書き保存する。
  - バックエンドでは、既にサーバーに導入されている Python ＋ PyMuPDF (fitz) を用いて、高品質な網膜ディスプレイ（Retina）対応のレンダリング処理を実行する。
- **バックアップとロールバック（安全性）**:
  - アップロードおよび変換処理の開始前に、現在の `public/images/map_base.png` を一時的にバックアップ（`map_base_backup.png`）としてコピーする。
  - 変換処理中に何らかのエラー（PDFの破損、Pythonプロセスの異常終了、ディスクフル等）が発生した場合、自動的にバックアップから復元（ロールバック）し、変更前の状態を維持する。
- **キャッシュ回避**:
  - ベースマップ画像が差し替えられた際、ブラウザの画像キャッシュによって古いマップが表示され続けるのを防ぐため、ビュー側で読み込む画像URLの末尾にアップロード日時のタイムスタンプ（例: `map_base.png?v=1782123850`）を付与する。
