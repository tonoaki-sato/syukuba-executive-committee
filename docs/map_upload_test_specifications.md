# ベースマップPDFアップロード・自動変換機能 テスト仕様書

本書は、「ベースマップPDFアップロード・自動変換機能 仕様書」および「同実装計画書」に基づき、該当機能を対象としたテストケースおよび検証手順を定義する。

---

## 1. テスト概要

本テストでは、ベースマップPDFのアップロード・画像自動変換処理における以下の観点を網羅的に検証する。

1. **アクセス制御（セキュリティ）**: 管理者（`admin`）のみが差し替え操作を行えること
2. **バリデーション**: PDF以外の拡張子やMIMEタイプ、10MB制限を超えるファイルの拒否
3. **正常画像変換**: 正当なPDFから1600px幅のPNG背景画像が正常生成され、差し替わること
4. **ロールバック（堅牢性）**: 変換エラー時（破損PDFなど）に以前の背景画像が完全復元されること
5. **キャッシュ回避**: アップロード完了後にブラウザキャッシュを迂回して新しい地図が表示されること
6. **UI/UX挙動**: アップロード中の進捗表示（二重送信防止ロック）および完了通知トースト

テストは、**「自動テスト（Laravel Featureテスト）」**と**「手動テスト（ブラックボックステスト）」**の双方を用いて実施する。

### 2.1 テストクラス
* **クラス名**: `tests/Feature/GozaichiMapUploadTest.php`
* **実行コマンド**: `php artisan test --filter=GozaichiMapUploadTest`

## 3. 手動テスト仕様（UI・運用検証）

### 3.1 テスト前提条件
- **テスト環境URL**: `https://www.syukuba.home`
- **検証用ダミーファイル**:
  - `valid_map.pdf`（正常なA3サイズ地図PDF、約1.2MB）
  - `invalid_file.png`（拡張子違いの画像、約500KB）
  - `corrupted.pdf`（壊れたPDFファイル、拡張子はpdfだが中身がテキスト）
  - `large_map.pdf`（12MBの極大PDFファイル）

### 3.2 手動テストシナリオ

#### シナリオ 1: 一般ユーザーのアップロード画面非表示確認
1. **一般会員**（または管理者権限のない**幹事**）でログインする。
2. ござ市地図配置画面（`/goza/map`）を開く。
3. **期待値**:
   - 画面内に「ベースマップ差し替え」ボタンやアップロード領域が**一切表示されないこと**。
   - 直接ブラウザの開発者ツール等から `POST /admin/map/upload-base` を叩いた場合、`403 Forbidden` となること。

#### シナリオ 2: 管理者画面でのファイル選択・バリデーション動作
1. **システム管理者**でログインし、ダッシュボードから地図配置画面を開く。
2. 設定ギアアイコンをクリックし、「ベースマップPDFの更新」モーダルを表示する。
3. **テスト A (拡張子バリデーション)**:
   - ドラッグ＆ドロップエリアに `invalid_file.png` をドロップする。
   - **期待値**: 「PDFファイル以外はアップロードできません」と画面に即時バリデーション警告が表示され、送信ボタンが非活性になること。
4. **テスト B (ファイルサイズバリデーション)**:
   - `large_map.pdf` を選択してアップロードを試みる。
   - **期待値**: 「ファイルサイズは10MB以下にする必要があります」と警告が表示され、アップロードが遮断されること。

#### シナリオ 3: 正常アップロードと自動差し替え・キャッシュ回避の動作
1. `valid_map.pdf` を選択し、「マップを更新する」ボタンをクリックする。
2. **期待値**:
   - アップロードおよび変換中、「ベースマップを変換中... (数秒かかります)」のローディングインジケータが表示され、画面全体の操作（クリックや再アップロード）がロックされること。
3. **期待値（処理完了後）**:
   - 「ベースマップの差し替えに成功しました」という成功トースト通知が表示されること。
   - モーダルが自動的に閉じ、背景画像が自動的に再読み込みされること。
   - 開発者ツールのネットワークタブで、背景画像（`map_base.png`）の末尾に `?v=1782123850` 等のタイムスタンプパラメータが付与され、ブラウザキャッシュを無視して最新の画像がサーバーから200ステータスで取得されていることを確認する。

#### シナリオ 4: 破損PDFアップロード時のロールバック確認
1. 正常なベースマップが表示されている状態で、`corrupted.pdf` を選択してアップロードを実行する。
2. **期待値**:
   - 変換エラーとなり、画面に「ベースマップの変換に失敗しました。以前のマップを維持します」とエラーメッセージが表示されること。
   - 画面のベースマップが、以前の正常なマップのまま崩れずに正しく描画され続けていること。
3. ストレージ内に `map_base_backup.png` などのゴミファイルが残っていないことをサーバーコンソールまたはログで確認する。
